# UVC (Upper Body Vertical Control) 算法详解

## 算法概述

UVC 是由 Dr. Guero 设计的一种专为舵机机器人优化的平衡控制算法。其核心思想极其优雅：

> **将 IMU 检测到的身体倾斜角，转化为支撑腿髋关节在横向(roll)和纵向(pitch)的位移调整量。**

与传统的 ZMP（零力矩点）或 MPC（模型预测控制）不同，UVC 不进行任何复杂的动力学计算。它本质上是一个**几何控制器**：通过简单的三角函数，将倾斜角映射为关节位移，再通过逆运动学转换为关节角度。

## 为什么 UVC 适合舵机？

1. **计算量极低**：只需要 `atan()`, `sin()`, `cos()` 和基本算术，可以在 STM32 上以 100Hz 运行
2. **位置控制友好**：输出的是位置增量，不是力矩——完美匹配舵机的控制模式
3. **低频鲁棒**：50Hz 控制频率即可工作，因为它是"积分式"控制，不依赖高频反馈
4. **容错性强**：积分 + 限幅 + 衰减的三段式设计，使系统不容易发散

## 算法流程

### 1. 死区处理

```python
k = sqrt(pitch² + roll²)  # 合成倾斜角
if k > 0.033:  # ~1.9°
    k1 = (k - 0.033) / k
    pitch *= k1; roll *= k1
else:
    pitch = 0; roll = 0
```

将死区范围内的微小倾斜忽略，避免在静态站立时产生无意义的修正。

### 2. 增益缩放

```python
rollt = 0.25 * roll    # 响应灵敏度 (0.25 ~ 0.85)
pitcht = 0.25 * pitch
```

较低的系数（0.25）使响应更保守但更稳定。较高的系数（0.85）响应更快但容易振荡。

### 3. 位移积分（核心）

```python
# Roll 方向修正
k = atan((dyi - sw) / autoH)    # 当前开腿角
kl = autoH / cos(k)             # 腿的实际长度
ks = k + rollt                  # 加入倾斜修正
dyi = kl * sin(ks) + sw         # 更新横向位移
autoH = kl * cos(ks)            # 更新腿高度
```

这是 UVC 最精妙的部分：它不是直接将角度映射为关节角，而是通过三角函数将角度变化**积分为位移量**。这使得控制具有"记忆性"——即使倾斜角恢复为 0，之前积累的位移修正仍然保留，直到被衰减机制逐步消除。

### 4. 限幅保护

```python
dyi = clamp(dyi, 0, 45mm)
dxi = clamp(dxi, -45mm, 45mm)
```

防止积分值溢出导致机械过载。

### 5. 衰减恢复

```python
# 着地期: 按剩余步数等比递减
k = dyi / (11 - fwct)
dyi -= k
dyis += k  # 转移给游脚

# 腿高度: 7% 指数恢复
autoH += (HEIGHT - autoH) * 0.07
```

在每个步态周期的着地期，积分值被逐步归零，为下一个周期做准备。

## 7 阶段状态机

| 阶段 | 编号 | 功能 | 持续时间 |
|------|------|------|---------|
| 初始化 | 710 | 缓慢移动到站立姿态 | ~1s |
| IMU 校准 | 720 | 100次平均值消除偏移 | ~2s |
| 倾斜监测 | 730 | 等待倾斜超过死区 | ∞ |
| UVC 主循环 | 740 | 单脚支撑，积分修正 | 20步 |
| 振动衰减 | 750 | 等待结构振动消散 | 30步 |
| 恢复步 | 760 | 另一条腿归位 | 20步 |
| 双脚站立 | 770 | 等待完全稳定 | 50步 |
